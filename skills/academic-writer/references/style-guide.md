# Полное руководство по стилю академического письма

Это руководство составлено на основе анализа 6 реальных дипломных работ (ВКР), которые прошли проверку антиплагиат-систем на ИИ-генерацию и были успешно защищены в ЯГТУ.

---

## Форматирование документа (docx)

При генерации docx через python-docx используй эти параметры (извлечены из эталонных работ):

| Элемент | Шрифт | Интервал перед | Интервал после | Выравнивание | Отступ 1-й строки | Примечания |
|---------|-------|---------------|---------------|-------------|-------------------|-----------|
| Основной текст | 14pt | 0pt | 0pt | По ширине | 1.25 см | |
| Заголовок 1 | 14pt, **без жирного** | 0pt | 36pt | По левому краю | 1.25 см | Новая страница |
| Заголовок 2 | 14pt, **без жирного** | 24pt | 24pt | По левому краю | 1.25 см | |
| Заголовок 3 | 14pt, **без жирного** | 12pt | 0pt | По левому краю | 1.25 см | Слитно с текстом |
| Подпись рисунка | 14pt | 6pt | 6pt | По ширине | 1.25 см | |
| Подпись таблицы | 14pt | 6pt | 6pt | По ширине | **1.25 см** | Отступ как у красной строки |
| Текст после таблицы | 14pt | 12pt | 0pt | По ширине | 1.25 см | Только 1-й абзац |

> **Критически важно:** Заголовок 3 (уровень 1.1.1) — НЕ жирный, отступ сверху 12 пт, снизу 0 пт.
> В python-docx: `s.font.bold = False`, `space_before=Pt(12)`, `space_after=Pt(0)`
>
> Подпись к таблице — `first_line_indent=Cm(1.25)` (как у абзаца основного текста), НЕ Cm(0).

Поля: левое 30мм, правое 10мм, верхнее 20мм, нижнее 20мм. Межстрочный интервал: одинарный. Шрифт: Times New Roman.

Для автособираемого оглавления: использовать встроенные стили Heading 1/2/3 в Word, затем вставить TOC-поле.

## 1. Банк разговорных вкраплений

Эти фразы -- главный инструмент создания "человеческого" текста. Используй 1-2 на подраздел, выбирая случайно:

### Переходы и связки
- "Ну а к минусам относят:"
- "И на это есть ответ."
- "...и вот почему."
- "А ещё в отличие от сторонних платформ..."
- "Что же с логистикой и транспортировкой?"
- "Хотя некоторые ошибочно и полагают, что..."
- "...ничего своего придумывать не надо."

### Оценочные
- "...довольно удобно."
- "Да и в целом точность часто оставляет желать лучшего."
- "Конечно, готовые решения имеют свои недостатки..."
- "Ещё одним безоговорочным плюсом будет..."
- "Это является очень неудобным решением."
- "...что является несомненным плюсом данного решения."

### Обращения и "мы"
- "Мы видим, что в представленном выше рисунке..."
- "...выбранная нами платформа..."
- "В нашем случае компьютер..."
- "Перейдем к рассмотрению..."
- "Рассмотрим плюсы и минусы."

### Неформальные глаголы
- "Клиент, решивший использовать готовые решения, наткнётся на 1С."
- "...пользователь натыкается на диаграмму..."
- "...выбор пал на PyCharm..."
- "...взор руководителей падает на..."

### Разговорные частицы и слова
- "довольно" (вместо "достаточно")
- "зачастую"
- "нужно/нужен" (вместо "необходимо")
- "Тут находятся..."
- "по типу"
- "по сути"
- "всего лишь"
- "некий/некую"

### IT-жаргон без перевода
- "фидбек" (вместо "обратная связь")
- "промт/промпты"
- "парсер", "парсинг"
- "фейки"
- "опенсорс"
- "бэкенд", "фронтенд"

## 2. Банк академических оборотов

### Целеполагание
- "Целью данной выпускной квалификационной работы является..."
- "Целью данной работы являлась..."
- "Основные задачи включают в себя..."
- "Для достижения поставленной цели был произведен анализ..."

### Обоснование
- "Актуальность темы обусловлена тем, что..."
- "Актуальность данного проекта обусловлена необходимостью..."
- "...выбор данной СУБД оправдан тем, что..."
- "Исходя из плюсов и минусов всех подходов было принято решение..."

### Описание выполненной работы
- "В ходе выполнения выпускной работы проведен анализ..."
- "На основе полученных данных сформулированы требования..."
- "В результате выполнения выпускной квалификационной работы разработано..."
- "Для реализации выпускной квалификационной работы использованы различные инструментальные и программные средства, такие как..."

### Описание содержания глав
- "Первая глава данной работы посвящена..."
- "Во второй главе описывается..."
- "Третья глава содержит описание..."
- "В четвертой главе содержится..."

### Выводы
- "Исходя из выше написанного делаем вывод, что..."
- "Подводя итог..."
- "Таким образом..."
- "Таким образом, все задачи, которые были поставлены в данной работе, успешно выполнены."
- "Данные вычисления дают понимание..."

### Канцеляризмы
- "В ходе выполнения..."
- "осуществляет полный цикл производства..."
- "посредством внешних ключей"
- "является весьма актуальной"
- "не представляется возможным"
- "Внедрение повлечёт за собой повышение эффективности..."

## 3. Банк связок и переходов

### Высокочастотные (2-4 раза на подраздел)
- "Также" / "Так же" -- чередуй оба написания
- "Кроме того,"
- "Например,"
- "Помимо этого,"

### Среднечастотные (1-2 раза на подраздел)
- "Однако"
- "Так, например,"
- "При этом"
- "В свою очередь"
- "Ещё одним преимуществом можно назвать..."
- "Причём"

### Низкочастотные (1 раз на главу)
- "Стоит отметить"
- "Нельзя обойти вниманием"
- "Отдельного упоминания заслуживает"
- "По правде говоря,"
- "Бесспорно,"
- "Непременно необходимо рассказать о..."

### Переходные фразы к новому разделу
- "Перейдём к рассмотрению..."
- "Далее будет рассказано о..."
- "Далее будет рассмотрен прецедент..."
- "Ниже представлены..."
- "Необходимо рассмотреть плюсы и минусы."

## 4. Шаблоны построения предложений

### Определение через тире
```
X -- это Y, который/которая/которое позволяет Z.
```
Примеры:
- "Внутренний сайт, также иногда называемый интранет-портал -- сайт, предназначенный исключительно для внутреннего пользования."
- "Redis (Remote Dictionary Server) -- это быстрое и эффективное хранилище данных в оперативной памяти с открытым исходным кодом."
- "Жизненный цикл -- это некая последовательность этапов, через которые проходит проект."

### Конструкция "не просто X, а Y"
```
X не просто [существительное], а [более сильное утверждение].
```
Примеры:
- "Мобильное приложение не просто преимущество, а необходимое условие для поддержания конкурентоспособности."
- "Внутренний сайт становится не просто удобным инструментом, а необходимостью."
- "Данные не просто инструмент для работы, а стратегический ресурс."

### Конструкция "для + существительное"
```
Для [описание задачи] используется/применяется X.
```
Примеры:
- "Для подобного проектирования используется UML."
- "Для разработки веб-ресурса понадобится специализированный фреймворк."
- "Для мобильной разработки принято решение использовать Flutter."

### Конструкция с "что позволяет"
```
X [описание], что позволяет Y.
```
Примеры:
- "Фреймворк обладает встроенной ORM, что позволяет взаимодействовать с базой данных с помощью Python-кода."
- "Система поддерживает контейнеризацию, что позволяет легко масштабировать приложение."

### Перечисление с "такие как" / "включая"
```
X предоставляет [набор возможностей], такие как: A, B, C и D.
```
Примеры:
- "...с множеством функций, таких как: подсветка синтаксиса, форматирование кода, автодополнение и автоматические варианты решения ошибок."
- "...включая добычу, обработку и реализацию древесины."

### Риторический вопрос + ответ
```
[Вопрос]? [Короткий ответ или переход к описанию].
```
Примеры:
- "Что же с логистикой и транспортировкой? И на это есть ответ."

## 5. Типичные стилистические шероховатости

Допускай 1-2 таких элемента на подраздел. Выбирай разные типы:

### Тип 1: Неидеальное словообразование
- "Исходя из выше написанного" (вместо "вышеизложенного")
- "не смотря на ограничения" (вместо "несмотря на")
- "Высоко уровневый фреймворк" (вместо "Высокоуровневый")

### Тип 2: Пропуск слова или лишнее слово
- "Плюсами такого решения:" (без "являются")
- "является достаточно удобным редактором" (лишнее "достаточно")
- "Бывает называют её диаграммой размещения" (пропущено "что")

### Тип 3: Тавтология и плеоназм
- "разработка стратегий разработки новых продуктов"
- "Необходимо необходимые данные..."

### Тип 4: Смешение стилей в одном предложении
- "Клиент, решивший использовать готовые решения, наткнётся на 1С."
- "...путём сарафанного радио и небольшими выездами по области."

### Тип 5: Повтор шаблонной фразы
- Использование одной и той же формулировки для описания каждого конкурента, каждой таблицы БД или каждой технологии

### Тип 6: Разговорная частица в академическом контексте
- "Ну а к минусам относят:" -- частица "ну"
- "...ну и активное сообщество разработчиков..."

## 6. Шаблоны описания технологий

Используй этот шаблон для каждой технологии в технологической части:

```
### 3.1.N [Название технологии]

[Название] -- [определение: что это такое, тип инструмента] [причастный оборот: "позволяющий/используемый для"]. [1-2 предложения о ключевых особенностях].

[Абзац о возможностях и преимуществах, 3-5 предложений. Включи перечисление через "такие как:". Используй "позволяет", "обеспечивает", "предоставляет".]

[Необязательный абзац: история создания или интересный факт. Например: "Python получил название отнюдь не от семейства змей, а от комедийного телешоу..."]

[Абзац о применении в проекте: "В данном проекте [название] используется для [конкретное применение]. Выбор пал на данное решение, поскольку [обоснование]." + ссылка [N]]

[Необязательно: "Ещё одним преимуществом можно назвать [что-то]." или разговорная вставка]
```

## 7. Шаблоны обзора конкурентов

Каждый конкурент описывается по похожему шаблону, но формулировки плюсов и минусов должны ВАРЬИРОВАТЬСЯ от конкурента к конкуренту, чтобы не создавать подозрений при проверке. Общая структура одинаковая, а слова -- разные.

```
### 1.4.N [Название системы]

[Название] -- [что это за система, для кого предназначена]. [1-2 предложения об основном функционале].

[Описание возможностей и тарифов, 2-3 абзаца]

[Рисунок N -- Интерфейс / Таблица тарифов]

[Плюсы -- РАЗНЫЕ формулировки для каждого конкурента:]
- Конкурент 1: "К преимуществам данного решения можно отнести:" + список
- Конкурент 2: "Среди достоинств платформы стоит выделить:" + список
- Конкурент 3: "Плюсами такого решения:" + список

[Минусы -- тоже РАЗНЫЕ формулировки:]
- Конкурент 1: "Ну а к минусам относят:" + список
- Конкурент 2: "Однако у решения есть ряд ограничений:" + список
- Конкурент 3: "Но не обошлось и без недостатков:" + список

[Завершающая фраза тоже варьируется:]
- "данные минусы являются весомыми для целевой аудитории проекта, чтобы отказаться от подобного решения."
- "перечисленные ограничения делают решение малопригодным для наших задач."
- "в совокупности эти недостатки не позволяют рассматривать платформу как основу для нашей разработки."
```

## 8. Шаблоны введения и заключения

### Введение

```
[Абзац 1: Контекст]
В эпоху цифровизации [отрасль/сфера] становится всё более [прилагательное]. [Компании/организации] стремятся [что делать], чтобы [зачем]. [Ещё 1-2 предложения о трендах].

[Абзац 2: Цель]
Целью данной выпускной квалификационной работы является [разработка/проектирование] [чего] для [кого/для чего].

[Абзац 3: Что сделано]
В ходе выполнения выпускной работы проведен анализ [чего], сформулированы [требования/задачи], выбраны [технологии/методы].

[Абзац 4: Актуальность]
Актуальность темы обусловлена тем, что [обоснование]. [1-2 предложения].

[Абзацы 5-7: Аннотация глав]
Первая глава данной работы посвящена [чему]. Во второй главе описывается [что]. Третья глава содержит описание [чего]. В четвертой главе содержится [что].
```

### Заключение

```
[Абзац 1]
Целью данной работы являлась [повтор цели из введения].

[Абзац 2]
Для достижения поставленной цели был произведен анализ [чего]. На основе полученных данных [были сформулированы / было выполнено] [что].

[Абзац 3]
В ходе работы [было разработано / была спроектирована] [что]. [Перечисление ключевых результатов].

[Абзац 4]
Таким образом, все задачи, которые были поставлены в данной работе, успешно выполнены.
```

## 9. Шаблоны выводов по главам

Каждая глава заканчивается разделом с выводом. Шаблон:

```
В данной главе [было проведено / был выполнен / были описаны] [что].

[Перечисление 3-5 пунктов сделанного, каждый начинается с глагола в прошедшем времени:]
- Был проведен анализ [чего].
- Были рассмотрены [что].
- Было принято и обосновано решение [о чем].
- Были сформулированы [что].
- Было выполнено [что].

[Завершающее предложение, связывающее с целью работы]
```

## 10. Описание интерфейса приложения

Описание интерфейса строится как "проведение по экранам" -- пошаговое описание действий пользователя:

```
При переходе на сайт пользователь попадает на страницу авторизации (рисунок N).

Авторизовавшись, [пользователь/работник] компании попадает на [главную страницу / страницу дашборда], на которой он может [отследить/просмотреть] [что] (рисунок N+1).

Если [пользователю] необходимо [получить больше информации / выполнить действие], он [нажимает/переходит] [куда].

[Описание конкретного экрана]. На рисунке N+2 показана [страница/форма/карточка] [чего].

Нажав на [элемент интерфейса], пользователь попадёт на страницу [чего], где может [что делать] (рисунок N+3).

Закончив работу на странице [чего], [пользователь] возвращается [куда] для [чего].
```

## 11. Описание таблиц базы данных

Для описания сущностей БД используются шаблонные фразы. Критически важно: одна и та же фраза повторяется для КАЖДОЙ сущности (так делают реальные студенты):

### Логическое проектирование (описание каждой сущности):
```
В таблице N описывается сущность «[Название]», а также ее атрибуты, первичный ключ и ссылки на другие сущности через внешние ключи. Сущность хранит информацию о [чем] в базе данных.
```

### Физическое проектирование (типы данных каждой сущности):
```
В таблице N обозначены ограничения и определены типы данных атрибутов сущности, что обозначена в логическом проектировании.
```

Эти фразы повторяются 10-15 раз -- для каждой таблицы. Это нормально и характерно для студенческих работ.

## 12. Антипаттерны

Чего НЕ делать -- это выдаст ИИ-генерацию:

### Лексические маркеры ИИ (НИКОГДА не используй)
- "В заключение стоит отметить, что..."
- "Важно подчеркнуть, что..."
- "Следует отметить, что..."
- "Необходимо отметить, что..."
- "В контексте данного исследования..."
- "В рамках данного исследования..."

### Структурные маркеры ИИ
- Все абзацы одинаковой длины
- Все предложения одинаковой длины
- Идеальная параллельная структура списков
- Каждый абзац начинается одинаково
- Симметричные конструкции "Во-первых... Во-вторых... В-третьих..."
- Точное одинаковое количество пунктов в каждом списке

### Стилистические маркеры ИИ
- Безупречная грамматика без единой шероховатости
- Отсутствие разговорных вкраплений
- Отсутствие повторов (ИИ старается не повторяться)
- Идеально сбалансированные аргументы "за" и "против"
- Текст звучит "как энциклопедия"
- Отсутствие авторского голоса и субъективных оценок
