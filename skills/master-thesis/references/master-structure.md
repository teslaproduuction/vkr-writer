# Типовая структура ВКР магистратуры (IT / Computer Science)

На основе реального эталона: магистерская диссертация российского вуза 2024 (111 стр., 96 источников, 4 научные публикации).

---

## Общие параметры

| Параметр | Бакалавр | Магистр |
|---|---|---|
| Объём | 60–80 стр. | 80–120 стр. |
| Источников | 30–50 | 60–100+ |
| Рисунков | 20–40 | 50–130 |
| Таблиц | 10–20 | 20–40 |
| Приложений | 1–3 | 3–7 |
| Публикации | необязательно | желательно (РИНЦ, ВАК) |

---


## Введение — детальная структура

Введение — самая формализованная часть; проверяется рецензентами в первую очередь.

### 1. Актуальность темы
Объём: 1–2 абзаца. Структура:
- Контекст (тренды в области, цифры, проблемы промышленности/науки)
- Нерешённые задачи
- Потребность

**Пример:**
> "Развитие технологий газотермического напыления в последние десятилетия привело к расширению области применения порошковых покрытий в авиационной промышленности [1, 2]. Вместе с тем точность прогнозирования свойств покрытий на лопатках газотурбинных двигателей остаётся недостаточной: существующие аналитические модели не учитывают нелинейный характер взаимодействия частиц с подложкой [3]."

### 2. Степень разработанности проблемы
Объём: 1–2 абзаца. 5–10 ссылок.

**Шаблон:**
> "Проблематика [темы] нашла отражение в трудах [авторы, годы]. В работах [автор, год] предложен подход к [задаче]; [автор, год] исследовал [аспект]. Тем не менее большинство работ ограничивается [условие / допущение], что не позволяет в полной мере [что сделать]."

### 3. Цель и задачи
**Цель:**
> "Целью настоящей работы является [разработка / создание / построение / исследование] [чего] для [применения]."

**Задачи** (5–7 штук, каждая — инфинитив):
1. провести анализ предметной области и существующих подходов к [теме];
2. разработать / построить математическую модель [чего];
3. предложить алгоритм / метод [чего];
4. реализовать программный комплекс / прототип на основе разработанных моделей;
5. провести вычислительный эксперимент и верифицировать полученные результаты;
6. оценить точность / эффективность разработанного решения;
7. [дополнительная задача по теме].

### 4. Объект и предмет исследования

> "**Объектом исследования** является [широкая область / система / технологический процесс]."
> "**Предметом исследования** выступают [конкретные свойства / характеристики / методы / алгоритмы] объекта."

**Правило:** объект шире предмета. Объект — то, в чём существует проблема. Предмет — конкретный аспект, который изучается.

### 5. Научная новизна (обязательно!)
3–5 конкретных пунктов. Формулировки начинаются с глагола:

- "Разработан(а) [метод / модель / алгоритм], отличающийся от известных [конкретным отличием]."
- "Предложен подход к [задаче], учитывающий [фактор], ранее не рассматривавшийся в литературе."
- "Получены аналитические зависимости [чего от чего], позволяющие [применение]."
- "Установлено, что [закономерность / явление / факт]."

### 6. Теоретическая значимость
> "Теоретическая значимость исследования состоит в том, что разработанные [модели / методы / алгоритмы] дополняют / расширяют / уточняют представления о [предмете исследования] и вносят вклад в развитие [области]."

### 7. Практическая значимость
> "Практическая значимость работы определяется возможностью применения [результатов / разработанного ПО / методики] в [конкретная отрасль / предприятие / задача], что позволит [конкретный эффект: сократить время, повысить точность, снизить затраты]."

### 8. Апробация результатов (если есть)
> "Основные положения и результаты работы докладывались на [N-й Всероссийской / международной] научно-технической конференции «[название]» ([год, место]). По теме диссертации опубликовано [N] научных работ, в том числе [N] статьи в изданиях перечня ВАК."

### 9. Методология исследования
> "Методологическую основу работы составляют методы [перечисление через запятую: математического моделирования, численного анализа, конечных элементов, нейронных сетей, системного анализа, объектно-ориентированного программирования]."

### 10. Положения, выносимые на защиту
3–5 пунктов — краткие тезисы научных вкладов:
1. Математическая модель [чего], позволяющая [что].
2. Алгоритм [чего] с характеристикой [точность / скорость / преимущество].
3. Результаты вычислительного эксперимента, подтверждающие [что].

### 11. Структура работы
> "Диссертация состоит из введения, [N] глав, заключения, списка использованных источников ([N] наименований) и [N] приложений. Общий объём работы составляет [N] страниц, включая [N] рисунков и [N] таблиц."

---

## Реферат (аннотация)

Объём: 0.5–1 страница. Обязательные элементы:
- Объём работы: страниц, рисунков, таблиц, источников, приложений
- Ключевые слова (5–8 через запятую)
- Объект исследования
- Цель работы
- Результаты работы
- Область применения

---

## Структура глав: шаблоны выводов

Каждая глава заканчивается разделом "Вывод по [название] части":

```
### N.M Вывод по [аналитической / теоретической / практической / проектно-технической] части

В данной главе [был проведён / было выполнено / были рассмотрены] [что].

[Нумерованный или маркированный список 4–6 пунктов — что конкретно сделано:]
— Проведён анализ [чего]; выявлено, что [результат].
— Рассмотрены существующие подходы к [задаче]; установлено, что [вывод].
— Разработана / построена [модель / методика / алгоритм].
— Сформулированы требования к [чему].
— Выбран инструментарий для [чего] и обосновано данное решение.

[Связь с целью работы:]
Полученные результаты служат основой для [следующего этапа / практической части].
```

## Диаграммы в магистерской (проектная часть)

В IT-магистерской диаграммы создаются через PlantUML (см. diagrams-guide.md из скила academic-writer).

Типичный состав для IT-магистерской:
1. **Диаграмма вариантов использования** — пользователь + системные прецеденты
2. **Диаграмма последовательности** — 2–3 ключевых сценария
3. **Диаграмма деятельности** — основной рабочий процесс или алгоритм
4. **Диаграмма компонентов или архитектуры** — для сложных систем
5. **Диаграмма классов** — основные классы (не перегружать, 5–10 классов)
6. **Диаграмма развёртывания** — инфраструктура

Концептуальная + логическая модели БД — по шаблонам PlantUML.
Физическая модель — таблицами в тексте (не диаграммой).
